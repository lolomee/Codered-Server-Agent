01

Overview

CodeRed Server Agent is a modular endpoint monitoring and security agent that collects logs, monitors files, detects threats, and reports to your CodeRed Manager. It ships with a custom CLI that makes configuration simple for any customer — no manual config file editing required.

CapabilityDescription
log-collectionSystem, application, and security log collection
fimFile Integrity Monitoring with SHA-256 checksums
inventoryHardware, OS, packages, processes, open ports
threatRootkit scanning, brute-force, MITRE ATT&CK detection
vulnCVE mapping for installed packages (NVD feed)
complianceCIS Benchmark and custom SCA policy checks
active-responseAuto-block IPs, kill processes, quarantine files on alert

03

Installation

ℹ️ Use the Linux installer (.sh) on Linux only. Use the Windows installer (.ps1) on Windows. They download completely separate scripts for each OS — they are not interchangeable.

Linux (Ubuntu / Debian / CentOS / RHEL / Fedora)

Open a terminal and run as root. Use the bash <(curl ...) syntax — do not use curl | bash as it breaks interactive prompts:

bash — recommended (manager IP pre-set)
CODERED_MANAGER_IP=103.13.123.89 bash <(curl -s https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/install-codered-agent.sh)
⚠️ Always use CODERED_MANAGER_IP=... bash <(curl ...) — never curl ... | bash. When piped, bash reads the script from stdin, so interactive read prompts pick up script lines instead of your keyboard input, corrupting the manager IP in the config.

The installer: detects distro → installs agent package → sets manager IP → auto-fixes log formats → ensures valid XML → installs codered-agent CLI → downloads Linux-specific templates and discovery engine → clears old agent keys → starts service.

Windows (Server 2016+ / Windows 10+)

Open PowerShell as Administrator and run each line separately:

powershell — step 1: download
iwr https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/install-codered-agent.ps1 -OutFile install-codered-agent.ps1
powershell — step 2: run
Set-ExecutionPolicy Bypass -Scope Process -Force
.\install-codered-agent.ps1

The installer: checks for Python (auto-installs if missing) → downloads agent MSI → installs silently with manager IP → installs Windows-specific CLI and discovery engine → downloads Windows templates → starts service.

⚠️ Do not use curl on Windows — it is an alias for Invoke-WebRequest. Do not pipe with | iex as antivirus (Bitdefender, Defender) may block it. Always download first with iwr ... -OutFile, then run locally.

Modules

Required
Log Collection
codered-agent enable log-collection
Collects logs from system files and application sources. Use codered-agent scan to manage included log paths.
Recommended
File Integrity Monitoring
codered-agent enable fim
Monitors /etc, /bin, /sbin for unauthorised changes using SHA-256 checksums. Detects create, modify, and delete events.
Recommended
System Inventory
codered-agent enable inventory
Snapshots installed packages, running processes, open ports, and network interfaces every hour.
Recommended
Threat Detection
codered-agent enable threat
Rootkit and hidden process scanning. Includes rules for SSH/web brute force, MITRE ATT&CK techniques, and privilege escalation.
Optional
Vulnerability Detection
codered-agent enable vuln
Compares installed packages against the NVD CVE database. Supports apt, yum, dnf, and Windows Update.
Optional
Compliance & Auditing
codered-agent enable compliance
Runs CIS Benchmark SCA checks. Custom YAML policies can be added to /etc/codered/sca/.
Optional
Active Response
codered-agent enable active-response
Automatically executes firewall drops and process kills when alerts are triggered. IPs are auto-unblocked after 5 minutes by default.
⚠️ Active Response caution: Enable only after testing in your environment. Misconfigured rules may block legitimate traffic including your own management connections.
07

Log Discovery

The log discovery engine automatically detects your OS and scans for relevant logs and event channels. It covers both Linux log files and Windows Event Log channels out of the box.

How It Works

1
Service Detection

Linux: queries systemctl and dpkg/rpm.
Windows: queries sc query and wmic product get.

2
Log Source Scan

Linux: checks 28+ known log paths under /var/log and scans for custom .log files.
Windows: verifies 12 Event Log channels via wevtutil and scans C:\inetpub\logs for IIS and custom logs.

3
Priority Scoring

Each source is scored High / Medium / Low by security relevance. High and Medium are pre-selected by default.

4
Config Injection

Selected sources are written as <localfile> blocks into the agent config and the agent is restarted.

Priority Levels

🟢 LowNo

Troubleshooting

Linux — Wrong manager IP / ossec.conf corrupted after install

If you ran curl ... | bash instead of bash <(curl ...), the manager IP gets set incorrectly and breaks ossec.conf. Fix it:

bash
# Fix manager IP and XML in one command
python3 -c "
import re, shutil
conf = '/var/ossec/etc/ossec.conf'
with open(conf) as f: content = f.read()
shutil.copy2(conf, conf + '.bak')
content = re.sub(r'<address>.*?</address>', '<address>103.13.123.89</address>', content)
content = content.replace('</ossec_config>', '').rstrip() + '\n</ossec_config>\n'
with open(conf, 'w') as f: f.write(content)
print('Fixed')
" && systemctl start wazuh-agent

Agent not connecting to manager

bash
# Verify manager IP
grep "<address>" /var/ossec/etc/ossec.conf

# Test connectivity
nc -zv 103.13.123.89 1514
nc -zv 103.13.123.89 1515

# Check agent log
tail -30 /var/ossec/logs/ossec.log

Agent service not starting

bash
systemctl status wazuh-agent --no-pager
journalctl -xeu wazuh-agent --no-pager | tail -30
sudo codered-agent restart

Module changes not taking effect

Always run restart after manual config edits:

bash
sudo codered-agent restart

Config file corrupted

A backup is created automatically before every change:

bash
sudo cp /var/ossec/etc/ossec.conf.bak /var/ossec/etc/ossec.conf
sudo codered-agent restart

Log discovery shows no logs

bash
# Check log directory
ls -lh /var/log/

# Check active services
systemctl list-units --type=service --state=active

# Re-run discovery
sudo codered-agent scan

Windows — ExecutionPolicy error

If you see "cannot be loaded because running scripts is disabled", run these as two separate commands:

powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
.\install-codered-agent.ps1
⚠️ Do not put both commands on the same line — PowerShell will throw a PositionalParameterNotFound error. Run each line separately.

Windows — Script not found error

If you see "is not recognized as the name of a cmdlet", PowerShell is running from the wrong directory. Navigate to where the script was downloaded first:

powershell
cd $env:USERPROFILE\Downloads
.\install-codered-agent.ps1

Windows — Antivirus blocking install

If Bitdefender or Windows Defender blocks iwr ... | iex, download the script first and run it locally:

powershell
iwr https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/install-codered-agent.ps1 -OutFile install-codered-agent.ps1
Set-ExecutionPolicy Bypass -Scope Process -Force
.\install-codered-agent.ps1

Windows — Python not found

If codered-agent setup says Python was not found, install it first:

powershell
winget install Python.Python.3.12 --accept-package-agreements

Close and reopen PowerShell after installing Python, then retry codered-agent setup. Make sure Add Python to PATH was ticked during install.

Uninstalling

You can uninstall directly from the management console — no need to run a separate script.

Method 1 — Via Management Console (recommended)

linux
sudo codered-agent

Then select [5] Uninstall Agent from the main menu. You will be shown a list of everything that will be removed, and must type UNINSTALL to confirm before anything is deleted.

[5] Uninstall Agent 
  Warning: This will remove CodeRed Server Agent from this system.
  All agent configuration and state will be deleted.

  Items to be removed:
    • Agent service and binaries
    • /var/ossec/etc/ossec.conf
    • /etc/codered/
    • /usr/local/bin/codered-agent

  Type UNINSTALL to confirm, or press Enter to cancel: UNINSTALL

  Stopping agent service...
  ✔ Service stopped.
  Removing packages...
  Removing CodeRed files...

  ✔ CodeRed Server Agent has been completely removed.

Method 2 — Linux one-liner script

bash
curl -s https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/uninstall-codered-agent.sh | sudo bash

Method 2 — Windows script

Open PowerShell as Administrator:

powershell
iwr https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/uninstall-codered-agent.ps1 -OutFile uninstall-codered-agent.ps1
Set-ExecutionPolicy Bypass -Scope Process -Force
.\uninstall-codered-agent.ps1

Manual steps — Linux

bash
# Stop and disable service
sudo systemctl stop codered-agent
sudo systemctl disable codered-agent

# Remove package
sudo apt-get remove --purge codered-agent   # Debian/Ubuntu
sudo yum remove codered-agent               # CentOS/RHEL

# Remove CodeRed files
sudo rm -rf /etc/codered
sudo rm -f /usr/local/bin/codered-agent

Manual steps — Windows

powershell
# Stop the service
Stop-Service -Name "Wazuh" -Force
Set-Service -Name "Wazuh" -StartupType Disabled

# Uninstall: Win + R → appwiz.cpl → find CodeRed Server Agent → Uninstall

# Remove leftover files
Remove-Item -Recurse -Force "C:\Program Files\CodeRed" -ErrorAction SilentlyContinue

# Clean PATH
$p = [Environment]::GetEnvironmentVariable("Path","Machine")
$p = ($p -split ";" | Where-Object { $_ -notlike "*CodeRed*" }) -join ";"
[Environment]::SetEnvironmentVariable("Path", $p, "Machine")
💡 After uninstalling, close and reopen your terminal to clear any old PATH entries from the current session.
