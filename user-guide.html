01

Overview

CodeRed Server Agent is built on top of the Wazuh agent with a custom CLI that removes the need to manually edit configuration files. Install once, then use codered-agent commands to manage everything.

CapabilityDescription
log-collectionSystem, application, and security log collection
fimFile Integrity Monitoring with SHA-256 checksums
inventoryHardware, OS, packages, processes, open ports
threatRootkit scanning, brute-force, MITRE ATT&CK detection
vulnCVE mapping for installed packages (NVD feed)
complianceCIS Benchmark and custom SCA policy checks
active-responseAuto-block IPs, kill processes, quarantine files on alert

03

Installation

Linux

Run the one-line installer as root. You will be prompted for your CodeRed Manager IP if not set.

bash 
curl -s https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/install-codered-agent.sh | sudo bash

To pass the Manager IP directly for silent/scripted deployments:

bash 
CODERED_MANAGER_IP=10.0.0.1 bash <(curl -s https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/install-codered-agent.sh)

The installer will: detect your distro → add Wazuh repo → install agent → install codered-agent CLI → download module templates and log discovery engine → start service → prompt for log scan.

Windows

Open PowerShell as Administrator and run:

powershell 
Set-ExecutionPolicy Bypass -Scope Process -Force
iwr https://raw.githubusercontent.com/lolomee/Codered-Server-Agent/main/install-codered-agent.ps1 | iex

CLI Reference

All commands require root / Administrator privileges.

CommandDescription
scanScan endpoint for logs and choose which to monitor
setupInteractive module enable/disable wizard
statusShow all module states and agent service status
enable <module>Enable a specific module
disable <module>Disable a specific module
restartApply config changes and restart the agent
bash — examples
# Run log discovery
sudo codered-agent scan

# Run module setup wizard
sudo codered-agent setup

# Check current status
sudo codered-agent status

# Enable vulnerability detection
sudo codered-agent enable vuln

# Disable active response
sudo codered-agent disable active-response

# Restart agent after manual config changes
sudo codered-agent restart

06

Modules

Required
Log Collection
codered-agent enable log-collection
Collects logs from system files and application sources. Use codered-agent scan to manage included log paths.
Recommended
File Integrity Monitoring
codered-agent enable fim
Monitors /etc, /bin, /sbin for unauthorised changes using SHA-256 checksums. Detects create, modify, and delete events.
Recommended
System Inventory
codered-agent enable inventory
Snapshots installed packages, running processes, open ports, and network interfaces every hour.
Recommended
Threat Detection
codered-agent enable threat
Rootkit and hidden process scanning. Includes rules for SSH/web brute force, MITRE ATT&CK techniques, and privilege escalation.
Optional
Vulnerability Detection
codered-agent enable vuln
Compares installed packages against the NVD CVE database. Supports apt, yum, dnf, and Windows Update.
Optional
Compliance & Auditing
codered-agent enable compliance
Runs CIS Benchmark SCA checks. Custom YAML policies can be added to /var/ossec/etc/shared/codered-sca/.
Optional
Active Response
codered-agent enable active-response
Automatically executes firewall drops and process kills when alerts are triggered. IPs are auto-unblocked after 5 minutes by default.
⚠️ Active Response caution: Enable only after testing in your environment. Misconfigured rules may block legitimate traffic including your own management connections.
07

Log Discovery

The log discovery engine (codered-discover.py) is the most important first step. It ensures you monitor the logs that matter for your specific environment.

How It Works

1
Service Detection

Queries systemctl for all active services and dpkg/rpm for installed packages.

2
Log File Scan

Checks 30+ known log paths, resolves globs, and scans /var/log for any custom .log files.

3
Priority Scoring

Assigns High / Medium / Low priority based on security relevance. High and Medium are selected by default.

4
Config Injection

Writes selected paths as <localfile> blocks into ossec.conf and restarts the agent.

Priority Levels

PriorityExamplesAuto-selected
🔴 Highauth.log, syslog, audit.log, ufw.log, kern.logYes
🟡 Mediumnginx/access.log, mysql/error.log, cron.log, dpkg.logYes
🟢 Lowmail.log, dmesgNo
ℹ️ Re-run sudo codered-agent scan whenever you install new services. The previous discovery configuration is automatically replaced.

Directory Structure

Linux

/usr/local/bin/
  └── codered-agent              ← CLI binary

/etc/codered/
  ├── state.json                ← enabled module state
  ├── codered-discover.py      ← log discovery engine
  └── templates/
      ├── log-collection.xml
      ├── fim.xml
      ├── inventory.xml
      ├── threat.xml
      ├── vuln.xml
      ├── compliance.xml
      └── active-response.xml

/var/ossec/
  ├── etc/
  │   └── ossec.conf            ← active agent config (managed by CLI)
  └── logs/
      └── ossec.log             ← agent operational log

Windows

C:\Program Files\CodeRed\Agent\
  ├── codered-agent.py          ← CLI script
  ├── codered-agent.cmd         ← CLI wrapper (run from anywhere)
  ├── codered-discover.py       ← log discovery engine
  └── templates\                 ← module config snippets

C:\Program Files (x86)\ossec-agent\
  ├── ossec.conf                ← active agent config
  └── logs\
      └── ossec.log

Troubleshooting

Agent not connecting to manager

bash
# Verify manager IP
grep "<address>" /var/ossec/etc/ossec.conf

# Test connectivity
nc -zv <manager-ip> 1514
nc -zv <manager-ip> 1515

# Check agent log
tail -f /var/ossec/logs/ossec.log

Agent service not starting

bash
systemctl status wazuh-agent
journalctl -u wazuh-agent -n 50
sudo codered-agent restart

Module changes not taking effect

Always run restart after manual config edits:

bash
sudo codered-agent restart

Config file corrupted

A backup is created automatically before every change:

bash
sudo cp /var/ossec/etc/ossec.conf.bak /var/ossec/etc/ossec.conf
sudo codered-agent restart

Log discovery shows no logs

bash
# Check log directory
ls -lh /var/log/

# Check active services
systemctl list-units --type=service --state=active

# Re-run discovery
sudo codered-agent scan

Uninstalling

Linux

bash
# Stop service
sudo systemctl stop wazuh-agent
sudo systemctl disable wazuh-agent

# Remove Wazuh agent
sudo apt-get remove --purge wazuh-agent   # Debian/Ubuntu
sudo yum remove wazuh-agent               # CentOS/RHEL

# Remove CodeRed files
sudo rm -rf /etc/codered
sudo rm -f /usr/local/bin/codered-agent

Windows

Go to Control Panel → Programs → Uninstall a program, find Wazuh Agent, and uninstall. Then delete C:\Program Files\CodeRed\ manually.
